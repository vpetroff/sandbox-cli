## Overview

The `execute` command allows you to run commands directly in your deployed sandbox environments without needing SSH or other remote access tools. It's perfect for debugging, testing, and managing your applications.

<Info>
  **Provider Support**: Currently available for Daytona with full real-time streaming. Azure ACI support is limited and E2B support is coming soon.
</Info>

## Basic Syntax

```bash
npm run cli execute "<command>" [options]
```

### Options

<ParamField path="command" type="string" required>
  The command to execute in the sandbox environment
</ParamField>

<ParamField path="--cwd" type="string" default="/workspaces/project">
  Working directory for command execution
</ParamField>

<ParamField path="--timeout" type="number" default="300">
  Command timeout in seconds
</ParamField>

<ParamField path="--no-stream" type="boolean" default="false">
  Disable real-time output streaming, return all output at once
</ParamField>

## Advanced Usage

### Long-Running Commands

For commands that take a long time to complete:

<CodeGroup>
```bash Extended Timeout
npm run cli execute "npm run build:production" --timeout 900
npm run cli execute "npm run test:e2e" --timeout 1200
```

```bash Background Processes
npm run cli execute "nohup npm start > app.log 2>&1 &"
npm run cli execute "screen -dmS myapp npm start"
```

```bash Monitoring Progress
npm run cli execute "tail -f build.log" --timeout 60
npm run cli execute "watch -n 5 'ps aux | grep node'" --timeout 120
```
</CodeGroup>

### Working Directory Management

<CodeGroup>
```bash Different Directories
npm run cli execute "ls -la" --cwd /tmp
npm run cli execute "ls -la" --cwd /var/log
npm run cli execute "ls -la" --cwd /workspaces/project/src
```

```bash Directory Navigation
npm run cli execute "pwd && ls -la"
npm run cli execute "cd /tmp && pwd && ls -la"
```
</CodeGroup>

### Output Control

<CodeGroup>
```bash Streaming Output (Default)
npm run cli execute "npm run build"
# Shows output in real-time as it happens
```

```bash Non-Streaming Output
npm run cli execute "cat large-file.json" --no-stream
# Returns all output at once when command completes
```

```bash Filtered Output
npm run cli execute "ps aux | grep node"
npm run cli execute "ls -la | grep '.js$'"
```
</CodeGroup>

## Provider-Specific Behavior

### Daytona Provider

<Check>**Full Support** with real-time streaming</Check>

**Features:**
- ‚úÖ Real-time output streaming
- ‚úÖ Working directory support
- ‚úÖ Command timeout handling
- ‚úÖ Signal interruption (Ctrl+C)
- ‚úÖ Exit code preservation
- ‚úÖ Both stdout and stderr capture

**Limitations:**
- Commands run with workspace user permissions
- Interactive commands requiring input may not work as expected
- Some system-level commands may be restricted
- Network access depends on workspace network policies

**Best Practices:**
- Use absolute paths or paths relative to `/workspaces/project`
- For long-running processes, consider using `screen` or `tmux`
- Test commands in Daytona's web terminal first if unsure

### Azure Container Instances

## Error Handling and Troubleshooting

### Common Errors

<AccordionGroup>
  <Accordion title="No current sandbox selected">
    **Error Message**: `‚ùå No current sandbox selected.`
    
    **Solution**:
    ```bash
    npm run cli select  # Choose an active sandbox
    # or
    npm run cli create <folder>  # Create a new sandbox
    ```
  </Accordion>
  
  <Accordion title="Provider does not support command execution">
    **Error Message**: `Provider X does not support command execution.`
    
    **Solution**:
    - Use Daytona provider for command execution
    - Check provider documentation for limitations
    - Wait for provider updates to support execution
  </Accordion>
  
  <Accordion title="Sandbox is not running">
    **Error Message**: `Sandbox is not running (status: stopped)`
    
    **Solution**:
    ```bash
    npm run cli list  # Check sandbox status
    # Restart sandbox through provider interface
    # Or create a new sandbox
    ```
  </Accordion>
  
  <Accordion title="Command timeout">
    **Error Message**: Command execution timed out
    
    **Solution**:
    ```bash
    # Increase timeout
    npm run cli execute "long-command" --timeout 600
    
    # Run in background
    npm run cli execute "nohup long-command &"
    ```
  </Accordion>
  
  <Accordion title="Permission denied">
    **Error Message**: `Permission denied` or `Access denied`
    
    **Solution**:
    - Check file permissions: `npm run cli execute "ls -la"`
    - Use appropriate user context
    - Try alternative commands
    - Check sandbox configuration
  </Accordion>
</AccordionGroup>

### Debugging Tips

<Tip>
  **Test commands locally first** to ensure they work before running in sandbox
</Tip>

<Tip>
  **Use `--no-stream` for commands with large output** to avoid overwhelming the terminal
</Tip>

<Tip>
  **Check exit codes** - commands that fail will exit with non-zero codes and show error output
</Tip>
